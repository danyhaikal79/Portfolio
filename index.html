Intelligent Poultry & Feed Monitoring System

Embedded Firmware + Edge AI · 42-Day Field Deployment

+

Two-tier edge-AI system that monitors poultry feed level and chicken count in real time. Field-deployed at UPM Serdang for 42 continuous days.

  • Planned the full system pipeline and execution schedule using Gantt charts.
  • Designed a two-tier architecture: ESP32 as a real-time sensor front-end reading DHT22 (temperature + humidity), with Jetson Nano + Pi Camera handling vision-based feed-level and chicken-count inference.
  • Resolved DHT22 bit-banged timing issues by writing ESP32 firmware in C/C++ (Arduino framework) that polls the sensor and streams clean readings to the Jetson Nano over UART.
  • Deployed the rig at the UPM Serdang study site for 42 continuous days, monitoring system health remotely every 2 days via SSH.
  • Annotated datasets with Roboflow, then trained and benchmarked TWO YOLO models (one for chicken counting, one for feed-level estimation), comparing YOLOv8, YOLOv11, YOLOv12, and YOLO26 to select the best per task.
  • Built the full IoT architecture using ThingSpeak for cloud telemetry and a Streamlit dashboard for the operator UI.
  • Validated model accuracy against a real-world weighing tool to ensure the system works beyond proof-of-concept.
ESP32 Jetson Nano DHT22 UART PyTorch YOLO Roboflow ThingSpeak Streamlit SSH

IoT CO₂ & MoldRisk Prediction System

LoRaWAN · LSTM · Industrial IoT

+

Industrial IoT pipeline that ingests environmental sensor data via a LoRaWAN gateway and predicts indoor CO₂ levels and MoldRisk 30 minutes ahead.

  • Deployed and configured an industrial IoT stack using Milesight environmental sensors, a Dragino LoRaWAN gateway, and ChirpStack network server software for device provisioning, network keys, and payload decoding.
  • Designed the full data pipeline in Node-RED so sensor readings (CO₂, pressure, humidity, temperature) flow from ChirpStack → MQTT → InfluxDB.
  • Collected sensor data daily for 7 continuous weeks to build a robust dataset for time-series model training.
  • Applied KNN-based imputation in scikit-learn to fill missing values, then engineered rolling-window features for the model.
  • Trained an LSTM model in Python to forecast CO₂ and MoldRisk 30 minutes ahead.
  • Exposed live inference through a Flask REST API and visualized predictions on a live Grafana dashboard.
LoRaWAN ChirpStack Dragino Milesight Node-RED MQTT InfluxDB LSTM Flask Grafana

RFID Smart Inventory Management System

Embedded Firmware · OOP · Full-Stack

+

Warehouse-grade RFID inventory tracker with real-time tag detection, cloud database sync, and an on-device touchscreen UI.

  • Led team architecture decisions (hardware selection, data-flow design, and UI optimization), ensuring the design was executable within hardware constraints.
  • Selected hardware: ESP-WROOM-32 microcontroller (dual-core Xtensa LX6, Wi-Fi + Bluetooth), PN532 NFC/RFID reader (13.56 MHz, ISO/IEC 14443), and an ILI9341 LCD touchscreen.
  • Developed embedded firmware in C/C++ (Arduino framework): PN532 driver over I²C, LCD driver over SPI, Wi-Fi sync client, and a non-blocking event loop for scan + UI events.
  • Structured the firmware with object-oriented programming to cleanly handle button placement, scan-event flow, and on-device data operations.
  • Built the full-stack backend (MySQL + REST API) + frontend to handle sort, add, delete, and scan operations with intuitive data movement on the touchscreen UI.
  • Evaluated and tested end-to-end including failure modes (lost Wi-Fi, duplicate scans, unknown tags).
ESP-WROOM-32 PN532 ILI9341 LCD Embedded C/C++ I²C SPI OOP MySQL REST API
